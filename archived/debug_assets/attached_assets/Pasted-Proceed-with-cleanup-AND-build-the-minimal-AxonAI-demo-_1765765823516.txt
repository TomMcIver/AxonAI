PHASE 2 — Minimal Demo Loop (end-to-end)
Goal: Student can do this in one session:
Login → choose class → chat tutor → generate 5-question quiz → submit answers → update mastery → see mastery on a simple dashboard view.

Implement these minimal components using EXISTING files if possible:
A) API endpoints (in api_routes.py):
   1) POST /api/tutor/chat
      input: {class_id, message}
      output: {reply, tokens_used}
      requirements:
        - enforce student_id from session
        - enforce class_id enrollment
        - enforce strict class+student scope in the prompt/system message

   2) POST /api/quiz/generate
      input: {class_id, topic(optional)}
      output: {quiz_id, questions:[{id, prompt, skill_tag, answer_type}]}
      implementation:
        - use quiz_builder_agent.py if possible; otherwise do a GPT-based generator
        - store quiz + questions in DB (use existing models if present; if not, create minimal tables)

   3) POST /api/quiz/submit
      input: {quiz_id, answers:[{question_id, answer, correct(optional)}]}
      output: {score, mastery_update}
      implementation:
        - if "correct" not provided, use GPT to grade short answers quickly (demo-friendly)
        - call mastery_tracking_agent.py to update mastery OR implement a minimal mastery update:
          mastery[skill_tag] = EMA update based on correct/incorrect

B) Pages (templates):
   - student_dashboard shows:
       - “Start chat” button
       - “Generate quiz” button
       - mastery table: skill_tag | mastery %
   - ai_tutor.html should support chat + quiz in the same page for a clean demo.

C) Profile summarizer (demo version):
   - create a simple function in ai_service.py or a new file services/profile_summarizer.py
   - it can be GPT-based:
     inputs: last N chat messages + mastery dict
     output: 5-10 line student profile summary
   - store summary in DB, update after quiz submit.
