Goal: Replace these non-AI parts with ML

A) Mastery Tracking (currently keyword heuristics) → Trainable Mastery Model

Implement a mastery estimator per (student, skill) that outputs a probability P(mastered).

Requirements
	•	Create a mastery_model/ module with:
	•	feature builder: from interaction + quiz attempts
	•	a baseline model (Logistic Regression or LightGBM/XGBoost if already used)
	•	calibration (optional but ideal)
	•	Features should include:
	•	attempt correctness history, rolling accuracy
	•	difficulty, time_taken, hint_usage (if available)
	•	engagement_score, message length, session cadence
	•	recency and frequency features
	•	Persistence:
	•	Add or use a table to store:
	•	MasteryState(student_id, skill, p_mastery, updated_at, model_version)
	•	Also persist trained model artifacts on disk: models/mastery/<version>/model.pkl
	•	Online learning:
	•	After every quiz/quick-check submission, update mastery state incrementally:
	•	either via an online-capable model (SGDClassifier/logistic) or
	•	via “mini-batch retrain” every N interactions per student/class
	•	Replace usages of the old heuristic mastery outputs with MasteryState.p_mastery.

B) At-Risk Detection (currently thresholds) → Risk Prediction Model

Implement a risk model that predicts P(at_risk_next_14_days).

Requirements
	•	Module: risk_model/
	•	Training data:
	•	derived labels from demo simulator initially (and later real outcomes)
	•	label definition configurable: e.g. predicted fail, score drop, disengagement
	•	Features:
	•	mastery slope (Δp_mastery), engagement trend, missed attempts, low accuracy streaks
	•	attendance_rate (if present), time-of-day patterns, topic churn
	•	Persistence:
	•	RiskScore(student_id, class_id, p_risk, top_drivers_json, updated_at, model_version)
	•	Explanations:
	•	Provide top drivers using:
	•	logistic coefficients OR
	•	SHAP if using tree models (only if already in dependencies)
	•	Replace rule-based at-risk output in teacher insights with this risk score.

C) Strategy Selection → Contextual Bandit (Adaptive)

Replace epsilon-greedy strategy selection with a contextual bandit that chooses strategies based on student context.

Requirements
	•	Keep epsilon-greedy as fallback; add feature flag USE_CONTEXTUAL_BANDIT.
	•	Implement contextual bandit:
	•	start with LinUCB or Thompson Sampling with linear reward model
	•	context vector includes mastery, risk, engagement, skill, difficulty, time-of-day
	•	Reward signal:
	•	primary: mastery gain Δp_mastery over next attempt window
	•	secondary: engagement change, correctness of next quick-check
	•	Persistence:
	•	BanditPolicyState table or JSON blob with:
	•	per-strategy parameters, counts, last_updated, model_version
	•	Online update:
	•	update policy after each observed reward event (quiz/quick-check outcome)

D) Retrieval (keyword overlap) → Embedding Retrieval (RAG-lite)

Replace keyword overlap retrieval with embeddings.

Requirements
	•	Module: retrieval/
	•	Create a simple vector store:
	•	compute embeddings for ContentFile/materials
	•	store in DB as arrays/blobs or a lightweight local index
	•	Query-time:
	•	embed user message
	•	return top-k content snippets by cosine similarity
	•	If no embedding model available, add a local small embedding model dependency OR use existing provider (whichever is easiest in this repo).
	•	Ensure retrieved content is injected into prompts exactly where old retrieval was used.
